**エス・エー・エス株式会社 GitHub運用ガイドライン**

## 📚 このディレクトリについて

本ディレクトリには、GitHub運用において素早く参照できるリファレンス資料を配置しています。
日常作業で頻繁に参照する内容や緊急時の対応手順など、実用的な情報をまとめた文書群です。

---

## 📑 ドキュメント一覧

### 🚀 [QUICK_REFERENCE.md](./QUICK_REFERENCE.md)
**クイックリファレンスガイド**

#### 概要
開発者が日常的に使用するGitHubコマンドや操作手順をコンパクトにまとめた早見表。

#### 主な内容
- **リポジトリクローン方法** - SSH/HTTPS/GitHub CLIでのクローン手順
- **日常作業フロー** - 朝一番から作業完了までの標準的な流れ
- **コミットメッセージ早見表** - タイプ別のコミットメッセージ例
- **Issue/PRラベル使い方** - 優先度・種別・状態・工数の各ラベル一覧
- **よく使うGitコマンド** - 基本コマンドとトラブルシューティング
- **GitHub CLI操作** - gh コマンドの活用方法
- **トラブルシューティング** - Permission denied、コンフリクト解決など
- **Tips & ベストプラクティス** - 作業効率化のコツ

#### こんな時に参照
- Gitコマンドを忘れた時
- コミットメッセージの書き方を確認したい時
- エラーが出て対処法を知りたい時
- 作業の流れを再確認したい時

---

### 🚨 [EMERGENCY_RESPONSE.md](./EMERGENCY_RESPONSE.md)
**緊急時対応マニュアル**

#### 概要
GitHub関連の緊急事態が発生した際の詳細な対応手順書。セキュリティインシデントからシステム障害まで幅広くカバー。

#### 主な内容
- **緊急度レベル定義** - L1〜L4の4段階レベルと対応時間
- **緊急時チェックリスト** - 初動確認から詳細確認までのステップ
- **セキュリティインシデント対応**
  - 認証情報の漏洩対応手順
  - リポジトリ誤Public化の対処
  - 不正アクセス検知時の封じ込め
- **システム障害対応**
  - 本番環境への誤デプロイのロールバック
  - 重要ブランチ破壊からの復旧
  - 大規模マージコンフリクトの解決
- **Hotfixフロー** - 緊急修正の作成からデプロイまで
- **緊急連絡網** - エスカレーションフローと連絡先一覧
- **事後対応** - ポストモーテム作成と再発防止策
- **予防措置** - 日常的なセキュリティ対策と自動化
- **定期訓練** - 机上訓練から総合訓練までのスケジュール

#### こんな時に参照
- APIキーやパスワードを誤ってコミットした時
- リポジトリを誤ってPublicにしてしまった時
- 不正アクセスの兆候を発見した時
- 本番環境に問題のあるコードをデプロイした時
- 重要なブランチを破壊してしまった時
- 緊急度の判定が必要な時
- インシデント後の報告書作成時

## 🔍 ドキュメントの使い分け

| 状況 | 推奨ドキュメント |
|-----|----------------|
| **日常のGit操作を確認したい** | QUICK_REFERENCE.md |
| **コマンドをすぐ知りたい** | QUICK_REFERENCE.md |
| **緊急事態が発生した** | EMERGENCY_RESPONSE.md |
| **セキュリティ問題が起きた** | EMERGENCY_RESPONSE.md |
| **エラーの対処法を探している** | QUICK_REFERENCE.md → トラブルシューティング |
| **Hotfixの手順を確認したい** | 両方（簡易版: QUICK_REFERENCE.md、詳細版: EMERGENCY_RESPONSE.md） |
| **連絡先を探している** | EMERGENCY_RESPONSE.md → 緊急連絡網 |
