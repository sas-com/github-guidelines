## 📚 ドキュメント一覧

このディレクトリには、GitHub運用における管理・運用に関する詳細なガイドラインが含まれています。各ドキュメントは特定の管理領域に焦点を当てており、実践的な手順と最適な方法を提供します。

## 📋 ドキュメントINDEX

### 1. [ISSUE_PROJECT_MANAGEMENT_GUIDE.md](./ISSUE_PROJECT_MANAGEMENT_GUIDE.md)
**Issue/Project管理ガイドライン**

GitHub IssuesとProject Boardsの効果的な活用方法を定義したガイドです。

**主な内容**：
- Issue ラベリング戦略（優先度、種別、ステータス、コンポーネント、技術スタック）
- Issue テンプレートの使い方（バグ報告、機能要望、セキュリティ、パフォーマンス）
- Project Board運用方法（カンバン方式、スプリント管理）
- マイルストーン管理とロードマップ
- 自動化とワークフロー最適化

**対象読者**：
- プロジェクトマネージャー
- 開発チームリーダー
- 全開発メンバー

---

### 2. [TEST_STRATEGY.md](./TEST_STRATEGY.md)
**テスト戦略文書**

包括的なテスト戦略とアプローチを定義した文書です。品質保証プロセス全体をカバーします。

**主な内容**：
- テストピラミッド戦略（ユニット70%、統合20%、E2E10%）
- 各テストレベルの詳細（ユニット、統合、E2E、パフォーマンス、セキュリティ）
- 自動テストフレームワークとツール
- CI/CDパイプライン統合
- テストカバレッジ目標と品質メトリクス
- セキュリティテスト戦略（SAST、DAST、依存関係チェック）

**対象読者**：
- QAエンジニア
- 開発エンジニア
- DevOpsエンジニア
- セキュリティチーム

---

### 3. [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)
**PR自動テスト トラブルシューティングガイド**

PR自動テストシステムで発生する問題の診断と解決方法を網羅したガイドです。

**主な内容**：
- 緊急度レベル分類（L1〜L4）
- 問題診断フロー（初期診断、ローカル再現、根本原因分析）
- カテゴリ別トラブルシューティング：
  - ビルドエラー対応
  - テスト失敗対応
  - 環境固有の問題
  - パフォーマンス問題
  - セキュリティスキャン問題
- 予防策とベストプラクティス
- エスカレーションプロセス

**対象読者**：
- 開発エンジニア
- DevOpsエンジニア
- サポートチーム
- インシデント対応担当者
